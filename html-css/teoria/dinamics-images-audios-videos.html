Imagens dinâmicas, audio e vídeo em HTML5

Visto que os conteúdos dos sites na internet podem ser acessados através de diversos tipos de aparelhos eletrônicos, que possuem diferentes tamanhos de display é necessário adaptar os arquivos de mídia para cada aparelho.

Imagens flexiveis

O primeiro passo para realizar a adaptação do conteúdo do site é gerar imagens de tamanhos diferentes (já que o site vai ser acessado por diferentes tipos de aparelhos) e fazer com que o navegador selecione e carregue a imagem que melhor cabe para tal display.
Para realizar tal tarefa é necessário fazer uso das tags <picture> e <source>. A tag <picture> permite que seja criada varias fontes de imagem para serem utilizadas. É importante lembrar que é necessário organizar as imagens de forma crescente de tamanho dentro da tag. Abaixo temos um exemplo de como fazer tal tarefa: 

            
                <picture>
                <source media="(max-width: 750px)" srcset="C:\Users\Gamer\learningToProgram\html-css\imagens\tamanho-p.png" type="image/png">
                <source media="(max-width: 1050px)" srcset="C:\Users\Gamer\learningToProgram\html-css\imagens\tamanho-m.png" type="image/png">
                <img src="C:\Users\Gamer\learningToProgram\html-css\imagens\tamanho-g.png" alt="imagem flexivel">
                </picture>

Áudio

Para reproduzir arquivos de áudios basta usar a tag <audio> companhada da tag <source> (novamente), assim o site vai ser capaz de reproduzir qualquer áudio. Importânte dar uma resalva sobre direitos autorais e o tamanho do áudio. Por fim, é possível (e recomendado) adicionar diferentes tipos do mesmo áudio, pois certos navegadores não reproduzem mp3 por exemplo, sendo assim com outro tipo de áudio (como wav(muito pesado) e ogg) após não reproduzir o primeiro, o navegador vai tentar reproduzir o próximo, por conta disso atente-se na ordem dos arquivos.
Segue abaixo um exemplo de como adicionar arquivos de áudio no site: 

            
                <audio preload="metadata" autoplay>
                <source src="Fix Bayonets - United States Marine Band.mp3" type="audio/mpeg">
                </audio>

Nesse exemplo temos outra maneira de reproduzir áudio, entretanto nem todos os navegadores conseguem reproduzi-lo.

Como a tentativa de reproduzir um segundo áudio acima pode falhar nesse navegador, por favor clique aqui para baixar o 2° áudio dessa página.

Como podemos ver dentro da HTML a tag <audio> é acompanhada de alguns novos atributos. O preload é um atributo que indica se o áudio será pré-carregado ou não recebendo 3 possíveis valores/respostas, sendo eles: 'metadata' (carrega apenas informações do arquivo, como tamanho, tempo, informações de direitos); 'none' (não carrega nada até o usuário clicar no play); 'auto' (é o padrão, que carrega o arquivo de áudio inteiro assim quando a página é carregada, mesmo que o usuário nunca aperte play).
Já o atributo 'controls' disponibiliza um player na tela. O atributo 'autoplay' por sua vez, inicia a reprodução do arquivo de áudio assim que a página é carregada. Por fim o atributo 'loop' repete o áudio eternamente enquanto o usuário estiver na página.

Vídeos

Para inserir vídeos no site, utilizamos a tag <video>, dentro dele podemos inserir uma tag já conhecida que é a <source>, os atributos 'controls', 'width', 'poster' e 'type'. Abaixo temos um exemplo: 

                
                    <video width="350" poster="p-m-g-images/cavalo-thumb-nail.jpg" controls>
                    <source src="Cavalinho-Video-webm.webm" type="video/webm">
                    <source src="Cavalinho-Video-1.m4v" type="video/mp4">
                    <source src="cavalinho-video.mp4" type="video/mp4">
                    <p>Caso você esteja vendo essa mensagem, seu dispositivo não consegue reproduzir o arquivo de vídeo.</p>
                    </video>
